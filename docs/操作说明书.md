# Quant Agent Lab Operator Manual

This manual is for the Ubuntu project at `/home/fc/projects/quant-agent-lab`.
The current system scope is A-share, daily / low-frequency, paper-only MVP.

## Environment

```bash
cd /home/fc/projects/quant-agent-lab
.venv/bin/python -c "import akshare, pandas, pyarrow, duckdb"
```

If `.venv` is missing, check `docs/ubuntu_runbook.md`. Do not install into
system Python.

## Current agent layout

```text
DataAgent -> RegimeAgent -> SignalAgent -> PositionAgent -> RiskAgent -> ExecutionAgent -> MonitorAgent
MetaAgent handles orchestration and downgrade logic
LLMReportAgent runs after report artifacts are written
LLMResearchAgent runs after offline research artifacts are written
```

## Real AKShare sync


## End-of-day pipeline

```bash
PYTHONPATH=src .venv/bin/python scripts/daily_pipeline.py --as-of 2025-01-31 --symbols 510300,510500 --lookback-days 5
```

If `--as-of` is a non-trading day, the pipeline falls back to the latest real
A-share trading day.

Output directory:

```text
runs/reports/YYYY-MM-DD/
  daily_summary.md
  daily_summary.json
  manual_orders.csv
  data_sync_report.json
  strategy_diagnostics.json
  llm_report.md
  llm_report.json
  llm_audit.jsonl
```

Checks:

- `daily_summary.md` includes regime, weights, risk action, reconcile status
- `daily_summary.json` includes `regime_health`, `meta_decision`, `data_sync`
- `strategy_diagnostics.json` includes symbol-level diagnostics
- `llm_report.json` is expected to show `status = skipped` by default
- `manual_orders.csv` is for manual review only, not auto execution

## Offline strategy research

Run deterministic research only:

```bash
PYTHONPATH=src .venv/bin/python scripts/strategy_research_run.py --use-simulated --symbols 510300,510500,600000 --end 2025-01-31
```

Enable research-side LLM overlay by setting `[llm.research_agent].enabled = true` in `configs/llm.toml`, then rerun.

Research output directory:

```text
runs/strategy_research/<timestamp>/
  config.json
  candidates.json
  metrics.json
  ranking.json
  strategy_diagnostics.json
  summary.md
  llm_research.md
  llm_research.json
  llm_audit.jsonl
```

Checks:

- deterministic artifacts always exist first
- if research LLM is disabled, `llm_research.json` does not exist
- if research LLM is enabled but provider is disabled, `llm_research.json` exists with `status = skipped`

## Forbidden actions

- Do not edit project files from the Windows workspace
- Do not connect a live broker API
- Do not let LLM generate orders or risk overrides
- Do not let Meta, Signal, or Position override Risk veto
- Do not treat `manual_orders.csv` as an auto-trading instruction
- Do not add Qlib, vectorbt, Backtrader, NautilusTrader, VeighNa, Kafka, or K8s to the current MVP baseline

## Runtime LLM prerequisite

Only consider runtime LLM after Stage 1-5 validation is stable, the end-of-day
pipeline runs cleanly for at least 5 trading days, and audit fields are settled.

本文档用于 Ubuntu `/home/fc/projects/quant-agent-lab` 当前版本。系统定位是 A 股日线/中低频/paper-only MVP，不接实盘券商，不自动下单。

## 1. 环境位置

```bash

```

如果 `.venv` 不存在，先看 `docs/ubuntu_runbook.md`。不要把依赖装进系统 Python。

## 2. 当前 Agent 架构

```text
DataAgent -> RegimeAgent -> SignalAgent -> PositionAgent -> RiskAgent -> ExecutionAgent -> MonitorAgent
MetaAgent 负责编排与安全降级
```

职责边界：`DataAgent` 做交易日历、数据同步和质检失败停止；`RegimeAgent` 输出市场状态和策略权重；`SignalAgent` 输出目标仓位；`PositionAgent` 把目标仓位转成 A 股订单意图；`RiskAgent` 是不能被覆盖的确定性 veto；`ExecutionAgent` 只执行已批准订单；`MonitorAgent` 生成日报；`MetaAgent` 编排和降级但不能直接创建订单或覆盖风控。

## 4. 真实 AKShare 数据同步

```bash
cd /home/fc/projects/quant-agent-lab
PYTHONPATH=src .venv/bin/python scripts/data_sync_akshare.py --start 2025-01-01 --end 2025-01-31 --symbols 510300,510500 --max-retries 2 --retry-backoff-seconds 1
```

失败处理：AKShare 网络或字段异常时必须失败，不要手工伪造数据文件；某个 symbol 失败时不要生成“成功”日报；交易日历不可用时不要用普通工作日替代 A 股交易日历。

## 6. 日终 Pipeline

```bash
cd /home/fc/projects/quant-agent-lab
PYTHONPATH=src .venv/bin/python scripts/daily_pipeline.py --as-of 2025-01-31 --symbols 510300,510500 --lookback-days 5
```

`2025-01-31` 是春节休市日时，系统会使用交易日历回退到最近一个 A 股交易日。

报告输出目录：

```text
runs/reports/YYYY-MM-DD/
  daily_summary.md
  daily_summary.json
  manual_orders.csv
  data_sync_report.json
```

检查重点：`daily_summary.md` 应包含 Regime、Regime weights、Risk action、Reconcile consistent；`daily_summary.json` 应包含 `regime_health`、`meta_decision`、`data_sync`；`manual_orders.csv` 只用于人工检查，不是自动下单文件。

## 8. 禁止事项

- 不在 Windows 工作区写本项目文件。
- 不接真实券商 API。
- 不用 LLM 生成订单、计算仓位、修改风控参数或触发下单。
- 不让 `MetaAgent`、`SignalAgent`、`PositionAgent` 覆盖 `RiskAgent` veto。
- 不把 `manual_orders.csv` 当成自动交易指令。
- 不引入 Qlib/vectorbt/Backtrader/NautilusTrader/VeighNa/Kafka/K8s 作为当前 MVP 依赖。

## 9. 运行时 LLM Agent 前置条件

只有在 Stage 1-5 验收全部通过、日终 pipeline 连续 5 个交易日稳定运行、`RiskAgent`/`PaperBroker`/日报/对账无重大偏差、并定义好 LLM 审计记录字段后，才进入运行时 LLM Agent 阶段。

LLM Agent 第一版只能做：日报解释、复盘草稿、异常标注、研究假设。它不能进入交易控制面。
